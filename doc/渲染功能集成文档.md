# PhysicsSync 渲染功能集成文档

## 概述

本文档描述了 PhysicsSync 项目中渲染功能的完整集成方案。渲染子系统基于 Google Filament 引擎，负责将物理模拟结果实时可视化，并提供用户交互接口。

## 架构设计

### 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    WinMain / Main                            │
│  (初始化渲染引擎 → 创建客户端 → 主循环 → 清理)               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    PhysicsClient                             │
│  (网络通信 · 物理模拟 · 客户端预测)                          │
│         │                                                   │
│         └── RenderCallback() ───────────┐                   │
└─────────────────────────────────────────┼───────────────────┘
                                          ▼
┌─────────────────────────────────────────────────────────────┐
│              RenderingPipeline (每帧)                        │
│  1. syncExistingFromSnapshot() - 同步物理状态                │
│  2. applyUpdates()           - 应用渲染变换                  │
│  3. renderFrame()            - 执行渲染                      │
└─────────────────────────────────────────────────────────────┘
```

### 核心组件

#### 1. FilamentRenderer - 渲染引擎封装

**职责**:
- 初始化 Google Filament 渲染引擎
- 管理渲染器、场景、视图的生命周期
- 处理窗口/交换链（通过原生句柄回调）
- 提供简化的帧渲染 API

**关键接口**:
```cpp
class FilamentRenderer {
public:
    // 初始化和销毁
    bool init(const RendererConfig& config, NativeWindowHandleCallback getWindowHandle);
    void destroy();

    // 场景管理
    bool createScene(const char* name = "MainScene");
    void destroyScene();

    // 相机管理
    bool createCamera(float aspectRatio, float nearPlane, float farPlane);
    void setCameraPosition(float x, float y, float z);
    void setCameraLookAt(float x, float y, float z);

    // 渲染循环
    bool renderFrame(int32_t w, int32_t h);

    // 查询接口
    filament::Engine* getEngine() const;
    filament::Scene* getScene() const;
    filament::Camera getCamera() const;
};
```

**设计要点**:
- 使用前置声明减少头文件依赖
- 通过回调函数获取窗口句柄，支持多平台适配
- 分离场景创建和视图创建，提高灵活性

#### 2. RenderingObjectManager - 渲染对象管理器

**职责**:
- 管理物理对象与渲染实体之间的映射关系
- 根据物理状态同步更新渲染实体的变换
- 管理材质和几何体
- 提供批量同步接口

**数据结构**:
```cpp
struct RenderObjectData {
    utils::Entity renderEntity;    // Filament 渲染实体
    void* renderable;               // 可渲染组件句柄
    void* transformInstance;        // 变换实例句柄
    Vec3 position;                  // 当前位置
    Quat rotation;                  // 当前旋转
    bool needsUpdate;               // 是否需要更新
};
```

**关键接口**:
```cpp
class RenderingObjectManager {
public:
    // 对象管理
    bool createObject(uint32_t objectId, const RenderObjectConfig& config);
    bool destroyObject(uint32_t objectId);
    bool hasObject(uint32_t objectId) const;

    // 物理状态同步
    void updateTransform(uint32_t objectId, const Vec3& position, const Quat& rotation);
    void syncFromSnapshot(const PhysicsWorldSnapshot& snapshot);
    void syncExistingFromSnapshot(const PhysicsWorldSnapshot& snapshot);
    void applyUpdates();

    // 查询
    Vec3 getPosition(uint32_t objectId) const;
};
```

**同步流程**:
1. `syncExistingFromSnapshot()` - 遍历物理快照，更新已有对象的变换
2. `applyUpdates()` - 将标记为需要更新的对象的变换应用到 Filament
3. 每帧渲染前调用 `applyUpdates()` 确保渲染对象与物理状态同步

#### 3. InputHandler - 输入处理器

**职责**:
- 捕获 Windows 键盘输入（WASD 键）
- 将按键状态转换为游戏输入
- 支持持续按键状态跟踪

**按键状态机**:
```
UP (初始/抬起)
  │
  ▼ (按键按下)
DOWN (刚按下)
  │
  ▼ (update() 转换)
HELD (持续按住)
  │
  ▼ (按键抬起)
UP
```

**关键接口**:
```cpp
class InputHandler {
public:
    bool initialize(const InputConfig& config);
    void update();
    void handleWindowsMessage(uint32_t msg, uint64_t wParam, int64_t lParam);
    
    PlayerInput getCurrentInput() const;
    bool isKeyDown(uint8_t key) const;
    bool isKeyJustPressed(uint8_t key) const;

    // 按键常量
    static constexpr uint8_t KEY_W = 0x57;
    static constexpr uint8_t KEY_A = 0x41;
    static constexpr uint8_t KEY_S = 0x53;
    static constexpr uint8_t KEY_D = 0x44;
};
```

**控制映射**:
- W/S: 前后移动
- A/D: 左右移动
- Shift: 奔跑

## 集成流程

### 1. 初始化顺序

```cpp
// 1. 初始化 Filament 渲染引擎
RendererConfig config(1280, 720, true);
FilamentRenderer renderer;
renderer.init(config, getWindowHandle);

// 2. 初始化渲染对象管理器
RenderingObjectManager renderMgr(renderer.getEngine(), renderer.getScene());
renderMgr.initialize();

// 3. 初始化输入处理器
InputHandler inputHandler;
inputHandler.initialize(inputConfig);

// 4. 创建并启动物理客户端
PhysicsClient client(config, renderCallback);
client.Initialize();
client.Start();
```

### 2. 渲染回调

```cpp
static void RenderCallback(uint32_t timestamp, float interpolationAlpha) {
    // 1. 获取物理世界状态
    PhysicsWorldSnapshot worldState;
    client.GetWorldState(worldState);
    
    // 2. 同步物理状态到渲染对象
    renderMgr.syncExistingFromSnapshot(worldState);
    renderMgr.applyUpdates();
    
    // 3. 处理输入
    inputHandler.update();
    PlayerInput input = inputHandler.getCurrentInput();
    
    // 4. 发送输入到服务器
    if (hasInput) {
        client.SendInput(input);
    }
    
    // 5. 执行渲染
    renderer.renderFrame(width, height);
}
```

### 3. 主循环

```cpp
while (running) {
    // 处理 Windows 消息
    if (PeekMessage(&msg, ...)) {
        if (msg.message == WM_QUIT) break;
        TranslateMessage(&msg);
        DispatchMessage(&msg);
        continue;
    }
    
    // 渲染帧
    renderer.renderFrame(width, height);
    
    // 统计信息
    printStatistics();
}
```

## 性能优化

### 1. 变换更新优化

使用 `needsUpdate` 标记，仅更新发生变化的对象：

```cpp
void RenderingObjectManager::applyUpdates() {
    for (auto& [objectId, data] : renderObjects_) {
        if (!data.needsUpdate) continue;  // 跳过未变化的对象
        // ... 更新变换
        data.needsUpdate = false;
    }
}
```

### 2. 批量同步

`syncExistingFromSnapshot()` 一次性处理所有对象的同步，避免逐帧频繁调用。

### 3. 渲染间隔

主循环中使用 `sleep_for(milliseconds(2))` 控制帧率，平衡 CPU 使用和渲染流畅度。

## 已知限制

### 1. 材质系统

当前材质系统为简化实现，使用占位方法：
- `createColorMaterial()` 返回 nullptr
- 实际使用需要预编译的 .matc 材质文件

### 2. 几何体创建

几何体创建函数为占位实现：
- `createBoxGeometry()`, `createSphereGeometry()`, `createPlaneGeometry()` 返回 true
- 实际使用需要构建自定义顶点缓冲区

### 3. 平台支持

当前实现主要针对 Windows 平台：
- 使用 WinMain 入口点
- 使用 Windows API 处理窗口消息
- Filament SwapChain 使用 HWND
