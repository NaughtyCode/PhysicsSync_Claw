# PhysicsSync 渲染功能实现文档

## 概述

本文档描述了为 PhysicsSync 项目集成的 Google Filament 渲染功能。该功能将 Jolt 物理引擎模拟的物体状态实时渲染到屏幕上，并提供 WASD 键盘控制玩家物体。

---

### 1.1 整体架构图

                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────┐

│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────────┐   │

### 1.2 数据流

1. **输入流**: WASD 按键 → InputHandler → PlayerInput → PhysicsClient → 服务器
2. **状态流**: 服务器快照 → PhysicsClient → RenderingObjectManager → 渲染对象
3. **渲染流**: 渲染对象 → TransformManager → GPU → SwapChain → 屏幕

## 三、核心组件说明

### 3.1 FilamentRenderer

**职责**: 封装 Google Filament 渲染引擎的初始化和基本操作。

**核心 API**:

```cpp
class FilamentRenderer {
public:
    // 初始化渲染引擎
    bool init(const RendererConfig& config, NativeWindowHandleCallback getWindowHandle);
    
    // 销毁渲染资源
    void destroy();
    
    // 场景管理
    bool createScene(const char* name = "MainScene");
    void destroyScene();
    filament::Scene* getScene() const;
    
    // 相机管理
    bool createCamera(float aspectRatio, float nearPlane, float farPlane);
    void setCameraPosition(float x, float y, float z);
    filament::Camera getCamera() const;
    
    // 渲染循环
    bool beginFrame(int32_t w, int32_t h);
    bool render();
    bool endFrame();
    bool renderFrame(int32_t w, int32_t h);
};
```

**关键设计**:
- 使用回调机制获取原生窗口句柄，实现平台抽象
- 支持窗口模式和离屏模式
- 生命周期管理：Engine → Renderer → Scene → View → SwapChain

### 3.2 RenderingObjectManager

**职责**: 管理物理物体与渲染实体之间的映射关系。

**核心 API**:

```cpp
class RenderingObjectManager {
public:
    // 对象管理
    bool createObject(uint32_t objectId, const RenderObjectConfig& config);
    bool destroyObject(uint32_t objectId);
    
    // 物理状态同步
    void updateTransform(uint32_t objectId, const Vec3& position, const Quat& rotation);
    void syncFromSnapshot(const PhysicsWorldSnapshot& snapshot);
    void applyUpdates();
    
    // 材质管理
    filament::MaterialInstance* createColorMaterial(float r, float g, float b, bool emissive);
};
```

**四元数到矩阵转换**:

```cpp
// 四元数 q + 位置 pos → Filament 矩阵
static void quatToMat4f(const Quat& q, float* outMat4f, const Vec3& pos);
// 输出格式：16 个浮点数，行主序
// | R00 R01 R02 Tx |
// | R10 R11 R12 Ty |
// | R20 R21 R22 Tz |
// |  0   0   0  1  |
```

**颜色映射**:

| 对象类型 | 颜色 | 用途 |
|---------|------|------|
| PLAYER | 绿色 (0,1,0) | 玩家控制物体 |
| DYNAMIC | 蓝色 (0.5,0.5,1) | 动态物理物体 |
| KINEMATIC | 黄色 (1,1,0) | 运动学物体 |
| STATIC | 灰色 (0.5,0.5,0.5) | 静态物体 |

### 3.3 InputHandler

**职责**: 处理 WASD 键盘输入，转换为游戏输入。

**核心 API**:

```cpp
class InputHandler {
public:
    bool initialize(const InputConfig& config);
    void update();
    
    // Windows 消息处理
    void handleWindowsMessage(uint32_t msg, uint64_t wParam, int64_t lParam);
    
    // 输入查询
    PlayerInput getCurrentInput() const;
    KeyState getKeyState(uint8_t key) const;
    bool isKeyDown(uint8_t key) const;
};
```

**按键映射**:

| 键 | 虚拟键码 | 功能 |
|----|---------|------|
| W | 0x57 | 向前移动 |
| S | 0x53 | 向后移动 |
| A | 0x41 | 向左移动 |
| D | 0x44 | 向右移动 |
| Shift | 0x10 | 奔跑（设置 sprint 按钮） |

**按键状态机**: `UP → DOWN → HELD → HELD → ... → UP`

## 四、构建与配置

### 4.1 CMake 配置

Filament 渲染库的配置是**可选的**：

```cmake
# 如果 external/filament 存在，则启用渲染
if(EXISTS ${EXTERNAL_DIR}/filament)
    set(RENDERING_AVAILABLE ON)
    # 配置头文件路径和库链接
endif()
```

编译时自动检测：
- **有 Filament**: 启用 `RENDERING_ENABLED` 宏，使用 `WinMain` 入口
- **无 Filament**: 使用 `main` 入口，仅输出文本

### 4.2 构建命令

```bash
# 配置（Release 模式）
cmake -B build -DCMAKE_BUILD_TYPE=Release

# 构建
cmake --build build --config Release

# 运行测试
ctest --test-dir build --config Release
```

### 4.3 运行

```bash
# 带渲染（需要 Filament 构建）
PhysicsSyncClientMain.exe --server 127.0.0.1 --port 9300

# 无渲染模式
PhysicsSyncClientMain.exe --help
```

## 五、渲染循环

### 5.1 每帧处理流程

```
┌─────────────────────────────────────────────────┐
│ 1. PhysicsClient 渲染回调 (每帧)                │
│    ├── 获取物理世界状态                          │
│    ├── RenderingObjectManager.syncFromSnapshot() │
│    ├── RenderingObjectManager.applyUpdates()    │
│    ├── InputHandler.update()                    │
│    └── InputHandler.getCurrentInput() → 发送    │
│                                                 │
│ 2. 主循环 (Windows 消息 + 渲染)                  │
│    ├── PeekMessage/DispatchMessage              │
│    └── FilamentRenderer.renderFrame()           │
└─────────────────────────────────────────────────┘
```

### 5.2 关键时序

```
帧 N:
  t=0ms    PhysicsClient 计算物理
  t=16ms   渲染回调: 同步状态 + 应用变换
  t=16ms   主循环: 渲染帧 → SwapChain → 屏幕
  t=32ms   下一帧...
```

## 七、已知限制与 TODO

### 7.1 当前限制

1. **材质系统**: 当前使用占位实现，需要预编译的 `.matc` 材质文件
2. **几何体**: 仅支持简单立方体/球体，高级几何体需要自定义顶点缓冲区
3. **光照**: 仅环境光，缺少动态光源
4. **输入**: 仅支持 WASD，缺少鼠标视角控制实现

### 7.2 后续改进

- [ ] 集成 Filament 的 `gltfio` 加载器支持 3D 模型
- [ ] 实现 PBR 材质系统
- [ ] 添加阴影和全局光照
- [ ] 实现鼠标视角控制
- [ ] 添加后处理效果（Bloom、SSAO 等）
- [ ] 支持多摄像机视图
- [ ] 添加调试可视化（碰撞体、边界框等）

## 十、快速参考

### 创建渲染对象

```cpp
RenderingObjectManager manager(engine, scene);
RenderObjectConfig config;
config.radius = 0.5f;
config.material = RenderMaterialConfig(1.0f, 0.0f, 0.0f);
manager.createObject(objectId, config);
```

### 同步物理状态

```cpp
// 从服务器快照同步
manager.syncFromSnapshot(serverSnapshot);

// 每帧应用更新
manager.applyUpdates();
```

### 控制玩家移动

```cpp
InputHandler handler;
handler.initialize();

// 在每帧
handler.update();
PlayerInput input = handler.getCurrentInput();
client.SendInput(input);
```
